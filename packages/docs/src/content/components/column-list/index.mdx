## Why use this component?

The column list component shows a list of items in a condensed format. It makes
use of the available space by breaking the list into a maximum of 2 or 3 columns,
depending on its configuration.

## Usage

Column lists fill the available horizontal space wherever they are inserted. They render
an `<ol>` element containing any number of child items.

The list will show in a single column on mobile, with the number of columns used
increasing as the viewport width increases, up to a maximum of 3 columns.

- The child elements passed into the column list must consist of at least one `<li>`
  element - no other elements are permitted
- The list items may contain links or plain text
- The standard variant of the column list uses 3 columns at the largest breakpoint
  and makes use of a boxed layout
- The boxed layout can be changed to a plain layout
- The maximum number of columns can be limited to 2 instead of 3
- The list can optionally be described with a heading, marked up using an
  `aria-labelledby` attribute on the column list component which corresponds to the
  heading's ID

## Code example

```
<h3 id="frequently-visited-topics">
	Topics most frequently visited by other users
</h3>
<ColumnList
	plain
	cols={2}
	aria-labelledby="frequently-visited-topics"
>
	<li>
		<Link to="/topics/hypertension/">Hypertension</Link>
	</li>
	<li>
		<Link to="/topics/diabetes-type-2/">Diabetes - type 2</Link>
	</li>
</ColumnList>
```

## Parameters

| Parameter name | Type    | Default | Description                                                                   |
| -------------- | ------- | ------- | ----------------------------------------------------------------------------- |
| plain          | Boolean | false   | Whether to render a plain list (as opposed to a boxed one)                    |
| cols           | Integer | 3       | The maximum number of columns to render. The only permitted values are 2 or 3 |

## Generated markup

The standard variant: 3 columns at the largest breakpoint, with a boxed layout.

```
<ol class="column-list column-list--boxed column-list--three-cols">
  <li>One</li>
  <li>Two</li>
  <li>Three</li>
  <li>Four</li>
  <li>Five</li>
  <li>Six</li>
</ol>
```

An alternative variant: plain rendering, with a maximum of 2 columns.

```
<ol class="column-list column-list--plain column-list--two-cols">
  <li>One</li>
  <li>Two</li>
  <li>Three</li>
  <li>Four</li>
  <li>Five</li>
  <li>Six</li>
</ol>
```
